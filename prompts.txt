PROMPT ENVIADO A COPILOT: 
Contexto:
"TechFix" es un servicio tecnico con mas de 8 años de experiencia atendiendo clientes particulares
y empresas. Ofrece diagnostico gratuito, reparaciones con garantia escrita y servicio a domicilio. El
sitio web debe transmitir profesionalismo y confianza tecnica, y permitir que los clientes soliciten un
turno de ingreso de equipo de forma rapida y sencilla.

Nuestro objetivo es Construir la pagina principal (home) del sitio de TechFix. El sitio debe transmitir profesionalismo
tecnico, comunicar claramente los servicios ofrecidos y guiar al visitante hacia el formulario de
ingreso de equipo.

Barra de Navegacion
- Nombre o logo del servicio tecnico a la izquierda.
- Links de navegacion: Inicio, Servicios, Precios, Nosotros y Ingresar equipo.
- El link "Ingresar equipo" debe apuntar a ingreso.html (pagina que se creara en la Etapa 2).
Seccion Hero
- Imagen de fondo relacionada con tecnologia o reparacion de computadores.
- Titulo principal con el nombre del servicio tecnico (etiqueta h1).
- Slogan que transmita confianza y rapidez.
- Boton principal "Ingresar mi equipo" que redirija a ingreso.html.
- Indicadores visuales de confianza (ejemplos: "Diagnostico gratuito", "Garantia escrita", "Mas
de 8 anos de experiencia"). Pueden ser tres iconos con texto corto debajo del hero.
Seccion de Servicios (minimo 6 servicios)
Mostrar los servicios que ofrece TechFix en formato de tarjetas. Ejemplos sugeridos:
- Reparacion de notebooks y computadores de escritorio
- Cambio de pantalla y teclado
- Limpieza y mantenimiento preventivo
- Instalacion y reinstalacion de sistema operativo
- Recuperacion de datos
- Soporte remoto
- Venta de repuestos y accesorios
Cada tarjeta debe incluir un icono o imagen, el nombre del servicio, una descripcion breve y un
tiempo de resolucion estimado (ejemplo: "Tiempo estimado: 24-48 hs"). Las tarjetas deben
organizarse en una grilla de al menos tres columnas en pantallas de escritorio.
Seccion Marcas y Dispositivos Atendidos
Mostrar las marcas o tipos de dispositivos que el servicio tecnico atiende. Minimo 6. Pueden
mostrarse como logos con nombre o como una lista estilizada con iconos. Ejemplos: HP, Dell,
Lenovo, Apple, Samsung, Asus, Acer, Impresoras Canon y Epson.
Seccion Tabla de Precios Referenciales
Mostrar una tabla HTML con precios referenciales de los servicios mas comunes. La tabla debe
incluir al menos 6 filas con las columnas: Servicio, Precio desde y Tiempo estimado. La tabla debe
estar estilizada con CSS (cabecera con color de fondo, filas alternadas, bordes).
Seccion Testimonios (minimo 3)
- Nombre del cliente, tipo de servicio recibido, texto del comentario y calificacion.
Footer
- Nombre y logo del servicio tecnico.
- Horarios de atencion presencial y remota.
- Direccion, email, telefono y WhatsApp ficticios.
- Links a redes sociales (pueden ser placeholders).
- Ano de copyright.
------------------------------------------------------------
RESPUESTA RECIBIDA (resumen breve): 
Página principal completada: hero (con CTA y 3 indicadores de confianza),
grid de servicios (7 tarjetas), marcas atendidas, tabla de precios (6 filas),
testimonios (3) y footer con contacto.

------------------------------------------------------------
USO EN EL PROYECTO: Se utilizó la respuesta tal y cual la entrego copilot, esto debido a que fue lo suficientemente
satisfactoria en calidad de homepage y style.
------------------------------------------------------------
REFLEXION PERSONAL: EL prompt fue útil para crear la estructura básica del proyecto, permitiendo ahorrar tiempo en su desarrollo,
en esta etapa no generaría muchos cambios, ya que estos pueden ser abordador a posterior.

============================================================ CONSULTA 5
Etapa del proyecto: Etapa 2 Fecha: 20-05-26
------------------------------------------------------------
PROMPT ENVIADO A COPILOT:
Ahora nuestro objetivo es el siguiente:
Crear la pagina ingreso.html con un formulario que permita al cliente registrar el ingreso de su
equipo al servicio tecnico. El formulario debe recopilar datos del cliente, informacion del dispositivo,
descripcion del problema y preferencias de contacto y entrega. Debe estar bien estructurado con
semantica HTML5 y presentar un diseno responsivo.
Debe también haber un archivo css llamado ingreso.css

El formulario debe estar dividido en cuatro secciones claramente diferenciadas de forma visual.
Seccion A — Datos del Cliente
- Nombre completo (input text, requerido).
- DNI (input text, requerido).
- Correo electronico (input email, requerido).
- Confirmar correo electronico (input email, requerido).
- Telefono de contacto (input tel, requerido).
- Tipo de cliente (radio buttons: Particular, Empresa). Si se elige "Empresa", mostrar un
campo adicional para el nombre de la empresa y el CUIT.
- Provincia (select con al menos 10 provincias argentinas, requerido).
- Localidad (input text, requerido).
Seccion B — Datos del Equipo
- Tipo de dispositivo (select: Notebook, Computador de escritorio, Impresora, Tablet,
Telefono movil, Otro, requerido). Si se elige "Otro", mostrar un campo de texto para
especificar.
- Marca (select con las marcas de la Etapa 1 mas la opcion "Otra", requerido). Si se elige
"Otra", mostrar campo de texto.
- Modelo (input text, requerido).
- Numero de serie o IMEI (input text, opcional, con nota aclaratoria de donde encontrarlo).
- Sistema operativo (select: Windows 10, Windows 11, macOS, Linux, Android, iOS, No
aplica / No sabe, requerido).
- El equipo tiene garantia vigente del fabricante (checkbox). Si esta marcado, mostrar campo
de texto para ingresar el numero de orden de compra o fecha de compra.
- El equipo contiene datos importantes sin respaldo (checkbox, con advertencia visible de
que TechFix no se responsabiliza por perdida de datos).
Seccion C — Descripcion del Problema
- Tipo de problema principal (select: No enciende, Pantalla danada, Lentitud o rendimiento,
Virus o malware, Problema de red o conectividad, Danio por liquido, Sonido, Otro,
requerido).
- Desde cuando ocurre el problema (select: Hoy, Hace algunos dias, Hace semanas, Hace
meses, requerido).
- El problema es permanente o intermitente (radio buttons: Permanente, Intermitente,
requerido).
- Descripcion detallada del problema (textarea, minimo 20 caracteres, maximo 500
caracteres, con contador, requerido).
- El cliente ya intento repararlo o lo llevo a otro tecnico (checkbox). Si esta marcado, mostrar
textarea para que describa que se hizo anteriormente (opcional, maximo 300 caracteres,
con contador).
Seccion D — Preferencias de Entrega y Confirmacion
- Modalidad de entrega del equipo (radio buttons: Lo llevo personalmente al local, Solicito
retiro a domicilio, requerido). Si se elige retiro a domicilio, mostrar campo de texto con la
direccion completa.
- Presupuesto maximo autorizado sin consulta previa (select: Hasta $5.000, Hasta $15.000,
Hasta $30.000, Sin limite, Necesito consultar antes de aprobar cualquier gasto, requerido).
- Preferencia de contacto para notificaciones (checkboxes: WhatsApp, Correo electronico,
Llamada telefonica). Al menos una debe seleccionarse.
- Horario preferido para ser contactado (select: Manana 9 a 12, Tarde 12 a 17, Tarde-noche
17 a 20, Indistinto, requerido).
- El cliente acepta que el diagnostico puede demorar hasta 48 horas habiles (checkbox
obligatorio).
- Acepta los Terminos y Condiciones del servicio (checkbox obligatorio, con enlace a pagina
ficticia).
Botones de Accion
- Boton principal "Registrar ingreso" (submit), centrado y con estilo destacado.
- Boton secundario "Limpiar" (reset).
- Enlace "Volver al inicio" que redirija a index.html.

Los requisitos de estilo y accesibilidad son los siguientes:
- Los campos deben cambiar visualmente al recibir el foco (:focus): borde con color y sombra
suave.
- Preparar las clases CSS "campo-error" (borde rojo) y "campo-ok" (borde verde) para la
Etapa 3.
- Las cuatro secciones deben estar visualmente separadas: fondo diferente, borde
redondeado o titulo de seccion destacado.
- El formulario debe estar centrado con un ancho maximo de 750 pixeles.
- Diseno responsivo: dos columnas en desktop, una columna en mobile.
- Todos los campos deben tener su etiqueta <label> asociada. Los requeridos deben
marcarse con asterisco.
- Los grupos de radio buttons y checkboxes deben estar dentro de <fieldset> con <legend>.
------------------------------------------------------------
RESPUESTA RECIBIDA (resumen breve):
"ingreso.html: 4 secciones organizadas, campos condicionales, validaciones, advertencias y botones.
ingreso.css: 4 secciones diferenciadas, paleta de colores, efectos en :focus, clase de validación, grid layout, formulario centrado
y tipografías."
------------------------------------------------------------
USO EN EL PROYECTO: Resultado bastante satisfactorio, se utilizó casi en su totalidad, ya que copilot se adelantó y creo el archivo
de validaciones JS sin antes tener contexto de qué se pediría exactamente en este.
------------------------------------------------------------
REFLEXION PERSONAL: El prompt fue útil, pero copilot se adelantó y creo el archivo JS sin decirle qué esperar en este, quizás
se podría especificar en el prompt que SOLO cree ingreso.html e ingreso.css, nada más.

PROMPT ENVIADO A COPILOT:
Ahora hay que implementar un archivo JS para hacer las validaciones, solo se debe utilizar JS nativo, nada externo, el nombre del archivo es "validaciones.js" el cual está vinculado a ingreso.html.

Objetivo:
Agregar el archivo validaciones.js que intercepte el envio del formulario, aplique todas las reglas de
validacion y proporcione retroalimentacion visual inmediata. Si el formulario pasa todas las
validaciones, debe mostrarse una pantalla de confirmacion de ingreso con un numero de orden
generado aleatoriamente.

Implementa, por ahora, validaciones solo para las siguientes 2 secciones:
Seccion A — Datos del Cliente
Campo Regla
Nombre completo Solo letras y espacios. Minimo 5 caracteres, maximo 80.
DNI Solo numeros. Entre 7 y 8 digitos.
Correo electronico Formato valido con @ y dominio. Usar expresion regular.
Confirmar correo Debe coincidir exactamente con el correo electronico.
Telefono Solo digitos, guiones, espacios y signo mas (+). Minimo 8 digitos
numericos.
Tipo de cliente Debe seleccionarse una opcion. Si es "Empresa", el nombre de
empresa y el CUIT no pueden estar vacios. El CUIT debe tener el
formato ##-########-# (con guiones) o 11 digitos numericos
seguidos.
Provincia y Localidad Provincia: seleccion obligatoria. Localidad: no puede estar vacia,
minimo 2 caracteres.
Seccion B — Datos del Equipo
Campo Regla
Tipo de dispositivo Seleccion obligatoria. Si es "Otro", el campo de texto no puede estar
vacio.
Marca Seleccion obligatoria. Si es "Otra", el campo de texto no puede estar
vacio.
Modelo No puede estar vacio. Minimo 2 caracteres.
Sistema operativo Seleccion obligatoria.
Numero de orden (garantia) Si el checkbox de garantia esta marcado, el campo de numero de
orden no puede estar vacio.

Solo de esas 2 secciones, nada más.
------------------------------------------------------------
RESPUESTA RECIBIDA (resumen breve):
"Se creo validaciones.js, con las secciones A y B, ingreso CSS recibió estilos para
.error-message, .confirmation-screen y retroalimentacion visual."

USO EN EL PROYECTO: El resultado es satisfactorio, muestra las comprobaciones en el formulario al dar Submit
------------------------------------------------------------
